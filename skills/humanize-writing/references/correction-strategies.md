# Correction Strategies

Proven techniques for fixing AI writing patterns. Organized by pattern type.

## Em-Dash Overuse (P1)

**Strategy:** Replace with the punctuation mark that fits the specific sentence.
- Parenthetical aside → use commas or parentheses
- Explanation following a statement → use a colon or period + new sentence
- Contrast or pivot → use "but", "though", or a period
- Attribution → restructure: "X, built with Y" instead of "X — built with Y"

**Do NOT:** Replace all em-dashes with semicolons (creates a new monotony pattern). Vary the replacements.

**Acceptable:** Keep 1-2 em-dashes if they genuinely serve the sentence. Zero em-dashes can look overcorrected.

## Buzzword Chains (P2)

**Strategy:** Explain in plain language what the thing actually does. Pretend you're describing it to a smart friend who doesn't know the jargon.

**Before:** "Zero-config semantic document search with AI-powered Q&A"
**After:** "Point it at a folder of documents and ask questions. It finds answers and shows you where they came from."

**Do NOT:** Remove all technical terms. Keep the ones that are genuinely descriptive (e.g., "FAISS", "Claude API"). Remove the marketing ones ("zero-config", "AI-powered", "semantic").

## Template Repetition (P4)

**Strategy:** Rewrite each bullet with a different structure.
- Start one with the achievement: "500,000 customers within 6 months."
- Start one with context: "A banking website I delivered..."
- Start one with first-person: "I built and ran..."
- Start one with commentary: "Not the fun kind of project, but..."

**Rule of thumb:** No two consecutive bullets should start with the same grammatical form.

## Parallel Structure (P5)

**Strategy:** Deliberately break the pattern mid-list. If the first 2 bullets start with verbs, make the 3rd start with a noun or context.

**Do NOT:** Make every item wildly different — that looks forced. Aim for natural variation, not chaos.

## Connector Monotony (P6)

**Strategy:** Same as P1. The fix is variety in punctuation, not replacement of one connector with another.

## Systematic Emoji (P7)

**Strategy:** Remove all or reduce to 1-2 meaningful ones. If keeping any, place them inconsistently (not every item).

**Better approach for most documents:** No emojis at all. Let the words do the work.

## Press Release Tone (P9)

**Strategy:** Rewrite from first person. Add context that only the actual person would know.

**Before:** "Accelerating digital initiatives during COVID-19"
**After:** "This was during COVID-19, so everything moved faster than usual"

**Key move:** Replace "what it sounds like to an outsider" with "what it felt like to the person doing it."

## Missing Personality (P10)

**Strategy:** Add 2-3 genuine asides. Not forced jokes — small honest observations.

Good examples:
- "Not the fun kind of project, but someone has to do it."
- "Still evolving." (shows work in progress)
- "I got tired of writing these by hand."
- "Not public yet, still getting it right."

**Do NOT:**
- Add personality to every line (overdone)
- Use catchphrases or memes
- Force humor where it doesn't fit
- Add self-deprecation about skills that matter (e.g., don't joke about being bad at coding on a developer profile)

## Absent First-Person Voice (P11)

**Strategy:** Convert product descriptions to personal accounts.

**Before:** "AI-assisted business rule and user story generator for agile teams."
**After:** "Generates business rules and user stories from plain-language input. I got tired of writing these by hand for every sprint."

**The test:** Would the author actually say this out loud? If not, rewrite.

## Vague Quantifiers (P12)

**Strategy:** Replace with actual numbers. If the number is unknown, either drop the quantifier or be honest: "a few" is better than "numerous."

**Before:** "across multiple organisations"
**After:** "at three organisations"
